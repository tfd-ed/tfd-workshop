Master the essential skills for working effectively in development teams. This series covers Git workflows, GitHub collaboration features, code review practices, and team communication strategies that are crucial for university projects and professional software development.

---

## 🎯 Series Overview

This series is designed specifically for **university students** and **junior developers** who want to learn how to:
- Collaborate effectively on code projects
- Use Git and GitHub like professional teams
- Manage tasks and track progress
- Review code and provide constructive feedback
- Navigate merge conflicts and team workflows

### Why This Series?

In today's software industry, technical skills alone aren't enough. The ability to work collaboratively, communicate effectively, and use modern development tools is essential. This series bridges the gap between individual coding skills and professional team development practices.

## 🚀 Getting Started

### Prerequisites

**Required:**
- Git installed locally
- GitHub account (free tier is sufficient)
- Basic command line knowledge
- Text editor (VS Code recommended)

**Recommended:**
- GitHub CLI (`gh`) for enhanced workflow
- VS Code with GitLens extension

### Setup Guide

1. **Verify Git installation:**
   ```bash
   git --version
   # Should show git version 2.x or higher
   ```

2. **Configure Git (if not done):**
   ```bash
   git config --global user.name "Your Name"
   git config --global user.email "your.email@example.com"
   ```

3. **Generate SSH key for GitHub (optional but recommended):**
   ```bash
   ssh-keygen -t ed25519 -C "your.email@example.com"
   # Follow prompts, then add to GitHub: Settings > SSH Keys
   ```

4. **Clone the workshop repository:**
   ```bash
   git clone https://github.com/KimangKhenng/tfd-workshop.git
   cd tfd-workshop/series/collab-dev
   ```

## 🎓 Learning Path

### For Complete Beginners
If you're new to Git, start here:
1. Complete a basic Git tutorial (e.g., [Git Handbook](https://guides.github.com/introduction/git-handbook/))
2. Practice basic commands: `init`, `add`, `commit`, `status`
3. Then proceed to Workshop 1

### For Those with Git Basics
If you know basic Git commands but haven't worked in a team:
1. Start with Workshop 1: Git & GitHub Team Workflow
2. Practice with the hands-on lab
3. Continue to Workshop 2 when available

### For Intermediate Users
If you've used Git in teams but want to improve:
1. Review Workshop 1 materials
2. Focus on advanced sections and best practices
3. Wait for Workshops 3-4 for advanced topics

## 💡 Workshop Philosophy

### Practical First
Every concept is taught through hands-on practice. You'll work on realistic scenarios that mirror real team projects.

### Collaboration Focus
Workshops emphasize team skills, not just technical knowledge. You'll learn communication, code review etiquette, and conflict resolution.

### Safe Learning Environment
Make mistakes in our exercises, not in production! Our labs are designed to let you experiment freely.

### Progressive Complexity
Each workshop builds on previous knowledge, gradually introducing more advanced concepts and techniques.
